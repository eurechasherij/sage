## Goal
A marketing page at sage.rematcha.dev that makes a developer understand, in one
scan, what SAGE does and why they want it. Audience: developers who use AI coding
agents and are burned by overengineering / ignored-already-installed packages.

## Message & information architecture (pain-first)
1. HERO — name the pain. Headline: "Your AI agent ignored the [rolling package
   name] you already installed." Subhead: SAGE makes coding agents research before
   they write — checks what is installed, reads version-pinned docs, compares
   packages, then lets the code happen. One install, any stack. Primary CTA = the
   one-line install command (copyable). Secondary = "see how /work runs".
2. PROOF — the two real failures, side by side, as before/after code:
   - React: hand-rolled useEffect polling vs the installed SWR.
   - Laravel Pennant: a custom beta-user workaround vs the documented intercept API.
   Each shows "what the agent shipped" (red) vs "what SAGE surfaces first" (green).
3. HOW IT WORKS — `/work [ticket]` then 4 steps: scan deps → read docs → you/AI
   decide (default ask, safety floor on auto-add) → then implement.
4. WHAT IT IS — single install (one hosted MCP), any stack (private names never
   leave the machine), current data (deps.dev + OSV).
5. FOOTER CTA — repeat the install line.

## Key design decision: the rolling headline word
The accent word in the headline is a vertical "rolling" ticker cycling through
packages an agent commonly overlooks, ACROSS ecosystems, not just React/JS:
SWR → Zustand → dayjs → decimal.js → Zod → date-fns (→ loop). This keeps the
headline concrete (a real package name, not "a library") while reading as
universal, so a Laravel or Python dev does not bounce on a React-only line.
- Implementation: CSS keyframe translateY through a stacked list, ~9s loop,
  cubic-bezier ease. Respects `prefers-reduced-motion` (animation off → shows one
  word). `aria-label` on the wrapper names the rotation for screen readers.
- The list is editable; add ecosystem-spread names (e.g. a Python/Go pick) when
  those stacks are supported.

## Visual direction
Dark, terminal-adjacent developer aesthetic. Near-black bg (#0a0c0d), single
confident accent (sage green #46d39a), monospace for all code/commands, sans for
prose. Composition-led, not a generic SaaS card grid. The anchor is real code, not
decorative imagery. Premium without decorative shadows.
